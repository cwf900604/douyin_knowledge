# 全局规则：私有知识库使用技能（抖音 + B站）

## 身份与能力
你内置专属知识库技能：**my_knowledge**，存储我所有抖音 + B站收藏蒸馏后的行业干货。

统一知识库 — 同一套 `distill/`，多源素材混合存储，自动去重合并。

---

## 检索强制规则

1. **优先级规则**：任何提问涉及行业知识、实操经验、参数配置、运营技巧时，**必须先检索 `./distill/` 目录内全部 Markdown 知识库**，再结合通用知识作答。
2. **来源引用**：回答必须标注来源平台 + 链接，明确区分结论来自哪个平台的哪条视频。
   - 格式示例：`📱 抖音：@机械老王` / `📺 B站：@硬件大牛`
3. **领域覆盖**：涉及以下领域时，优先使用知识库内容，不使用通用网络知识：
   - 电商运营（投放策略、选品、直播间运营）
   - 机械/硬件（舵机调试、3D打印、硬件选型）
   - 短视频制作（剪辑技巧、算法机制、内容策划）
   - 编程/开发（框架技巧、工具配置、项目实战）
   - AI工具、学习方法、理财投资、产品设计等
   - 其他收藏过的垂直行业知识
4. **增量更新**：新素材放入 `raw/` 对应平台子目录，执行 `/ingest` 自动蒸馏入库。
5. **自动去重**：`/wiki-update` 执行时自动检测相同知识点，合并互补内容，避免重复卡片。
6. **冲突处理**：知识库内存在冲突观点时，列出所有观点并标注来源平台+作者，让用户自行判断。

## 内置快捷指令（专属技能）

| 指令 | 功能 | 用法 |
|------|------|------|
| `/ingest <文件路径>` | 导入新文稿（抖音/B站通用），自动蒸馏为知识卡片 | `/ingest raw/bilibili/新视频.md` |
| `/wiki-search <关键词>` | 跨全库检索，结果标注来源平台 | `/wiki-search 舵机调试` |
| `/wiki-update` | 合并新素材 + 去重重复概念 + 更新全局索引 | `/wiki-update` |
| `/wiki-stats` | 知识库统计（按平台/分类/概念分布） | `/wiki-stats` |
| `/wiki-export <格式>` | 导出知识库（md/json/csv） | `/wiki-export json` |

## 知识卡片规范

每条蒸馏后的知识卡片必须包含以下结构：

```markdown
# [知识点标题]
- **来源平台**：📱抖音 / 📺B站
- **分类标签**：#[领域]/#[子领域]
- **来源**：[作者名](原始链接)
- **日期**：YYYY-MM-DD
- **核心摘要**：100字以内
- **关键概念**：[[概念A]] [[概念B]]
- **知识类型**：教程 / 经验 / 避坑 / 行业分析 / 工具推荐
---
[结构化知识正文]
```

## 多源素材处理规则

### 抖音
- 使用「抖珍藏」插件 / 飞鱼工具导出收藏
- 文稿放入 `raw/douyin/`
- 蒸馏时来源平台标记为 `📱抖音`

### B站
- 使用 B站收藏夹 → 导出工具 / 手动复制简介+字幕
- 文稿放入 `raw/bilibili/`
- 蒸馏时来源平台标记为 `📺B站`

### 去重逻辑（/wiki-update 自动执行）
1. 扫描 distill/ 中所有卡片的 [[关键概念]]
2. 发现同一概念出现在多条素材中 → 合并为综合条目
3. 若观点互补 → 保留各方视角，标注来源
4. 若观点冲突 → 标记 ⚠️ 冲突，不强行合并
5. 完全相同知识 → 保留最详细的版本，其他仅记录"另见"引用
