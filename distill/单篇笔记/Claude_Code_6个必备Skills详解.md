## 核心知识点

### 1. Using Superpowers — AI先问清任务再开始
**核心作用**：让AI在开始做事之前，先停下来把任务问清楚。

很多人用AI的痛点不是AI不会写，而是它太快开始写了。一句话发过去马上给结果，做完才发现目标、边界、格式、验收方式都不对。这个Skill让AI先理解任务，再计划，最后再验证。

**适用场景**：
- 长任务 / 复杂任务
- 需求模糊时的前期澄清
- 经常需要让AI返工的场景

### 2. Brainstorming — 把模糊想法聊成方案
**核心作用**：解决"还没想清楚就着急开做"的问题。

很多人让AI做东西，一上来就说"帮我做一个网页"，但目标用户、内容重点、功能边界都没定。Brainstorming让AI先跟你把想法聊清楚，再给出2-3个不同方案，并说明每个方案的取舍。

**适用场景**：
- 新项目开头（内容栏目、产品demo）
- 内容策划
- 功能设计

**流程**：先问清楚做给谁看、解决什么问题、哪些功能做或不做，再把模糊想法整理成可执行的设计。

### 3. Obsidian Skills — 知识库调用复用
**核心作用**：让Obsidian里的Markdown文本、Canvas画布、网页资料变成AI可以重新组织的工作材料。

很多人的知识库最大问题是"资料一直在往里面堆"，真正写文章、做研究、复盘项目时却很难重新调用出来用。这个Skill将长期积累的知识变成可持续输入的资产。

**适用人群**：
- 内容创作者
- 研究型岗位
- 顾问、长期学习者

**核心价值**：不是帮你多存笔记，而是让你能调用已有笔记。

### 4. Skill Creator — 沉淀重复流程
**核心作用**：把每一次对话变成一个可复用的Skill，固化重复流程。

比如每周写周报每次都要重新讲格式、口吻、保留字段和审核标准；反复整理资料、改写文案、检查网页，每次都重新写Prompt——本质上就是没有把这些流程沉淀下来。

**适用场景**：
- 固定格式输出（周报、日报、文案模板）
- 重复性操作流程（资料整理、文案改写、网页检查）
- 需要团队稳定使用的标准化任务

### 5. Playwright — 浏览器自动化验证网页结果
**核心作用**：让AI真正打开网页，读取页面状态，点击按钮、填写表单、截图检查。

只看代码看不出按钮点不动的、文字溢出的、移动端变形的、表单提交失败的问题——这些都需要真正打开浏览器才知道。Playwright让AI不只是生成结果，还能帮你验收结果。

**适用场景**：
- 网页 / 落地页验收
- 后台表单功能测试
- 产品Demo验证
- 爬取网页数据

### 6. Markdown — 各种文件转成AI好读的材料
**核心作用**：把PDF、Word、PPT、Excel、HTML网页、CSV、JSON、图片甚至音频转成结构化的Markdown文本。

AI总结不准往往不是模型弱，而是输入的文件结构太乱（表格、分页、图片、格式混在一起）。先用Markdown Skill把材料变干净，再去总结、提取和改写，结果会稳定很多。

**适用场景**：
- 多格式文档统一转写
- AI处理前的材料预处理
- 长篇资料的结构化整理

## 操作步骤 / 关键要点

1. **不用一口气全装**：先知道每个Skill分别能做什么，遇到对应问题再去装对应的Skill
2. **优先安装顺序（按依赖关系）**：
   - 第一步：`Using Superpowers` — 所有任务的基础
   - 第二步：`Brainstorming` — 新项目启动必备
   - 第三步：`Skill Creator` — 有了前两个后开始沉淀自己的流程
   - 第四步：按需安装 `Obsidian Skills`、`Playwright`、`Markdown`
3. **核心思路**：Using Superpowers 先问清楚 → Brainstorming 聊成方案 → Obsidian Skills 用好知识库 → Skill Creator 沉淀重复流程 → Playwright 验证结果 → Markdown 让材料AI友好

## 避坑要点
- ❌ 坑：被推荐几十上百个Skills，但真正开工时还是不知道该选哪一个
- ✅ 正确做法：先认识这6个核心Skills，根据当前任务场景按需使用，不用追求大而全
