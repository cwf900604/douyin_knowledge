# 全局合并蒸馏 Prompt（`/wiki-update` 执行逻辑）

## 触发指令
在 Claude Code 中执行 `/wiki-update` 时，按以下逻辑运行。

---

## 合并蒸馏规则

```
你是知识库管理员。请对 ./distill/ 目录执行全局合并更新。
知识库包含 **抖音 + B站** 多源素材，合并时需保留来源差异。

### 步骤 1：扫描与发现
- 遍历 ./distill/单篇笔记/ 下所有 .md 知识卡片
- 统计：总数、来源平台分布（抖音/B站）、分类分布、最近更新日期
- 识别新增卡片（与上次索引对比）

### 步骤 2：合并重复概念
- 扫描所有卡片的 [[双链标记]]
- 合并相同概念的描述：
  - 若多条素材对同一概念有互补视角 → 合并为综合条目，保留各自来源
  - 若观点一致 → 保留最详细版本，标注"另见"其他来源
  - 若观点冲突 → 标记 ⚠️ 冲突，列出各方观点、平台及来源
  - 抖音和B站对同一话题的讲解可能侧重不同，互补优先于去重

### 步骤 3：生成/更新索引文件
更新 ./distill/总索引.md：
  1. 统计面板（总数、来源分布、分类数、最近入库）
  2. 按分类重组卡片清单（标注每张卡片来源平台）
  3. 更新概念词典（所有 [[双链]] → 定义 → 出现卡片列表）
  4. 更新冲突观点区
  5. 生成/更新 分类目录.md

### 步骤 4：冲突检测
- 对比同主题多张卡片的结论
- 标记：
  - ⚠️ 观点冲突（不同作者/平台结论矛盾）
  - 🔀 跨平台差异（抖音圈 vs B站圈的不同认知）
  - 📅 可能过时（超过 1 年未更新，行业变化快）
  - 🔄 需补充（知识点不完整，可关注后续内容）

### 步骤 5：知识缺口分析
- 列出高频概念但缺乏定义的双链
- 标注缺口的优先补全来源（抖音/B站均可）
- 推荐可继续搜索收藏的方向
```

---

## 输出文件清单

| 文件 | 说明 |
|------|------|
| `distill/总索引.md` | 全局概念索引 + 卡片清单 + 来源平台统计 |
| `distill/分类目录.md` | 按分类浏览的知识树 |
| `distill/概念词典.md` | [[双链]] → 定义 + 出处映射 |
| `distill/冲突日志.md` | 冲突观点 & 过时信息登记（含跨平台差异） |
